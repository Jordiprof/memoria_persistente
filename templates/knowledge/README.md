# knowledge/ — Conocimiento duradero del proyecto

Aquí va lo que **no debe quedar enterrado en el journal**: hechos estables que querrás
re-consultar dentro de meses. Se consulta **on-demand** (no se lee en `/inicio`).

Crea archivos solo cuando haya contenido real (no dejes stubs vacíos). Sugerencias:

- `stack.md` — lenguajes, frameworks, versiones, servicios, comandos clave.
- `arquitectura.md` — estructura, módulos, flujos, diagramas.
- `convenciones.md` — estilo, naming, reglas del equipo, do/don't.
- `decisiones.md` — decisiones de arquitectura (ADR) duraderas y su porqué.

**Diferencia con el resto de la memoria:**
`STATE` = dónde estamos hoy · `TASKS` = qué falta · `journal` = qué pasó (cronológico) ·
`knowledge` = lo que es verdad de forma estable.
